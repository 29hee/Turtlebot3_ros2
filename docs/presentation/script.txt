# 1. 표지
안녕하세요. 발표를 맡은 이규희 입니다.
저희 팀은 ROS2에서 배운 SLAM, Nav2, OpenCV, OCR을 하나의 서비스로 연결해
자율주행 큐레이터 로봇을 구현했습니다.

기존 Nav2 예제들은 목표 좌표를 x,y를 통해 직접 입력해야 하지만
실사용자는 3,5좌표로 가! 라는 요구가 아닌 초록색 카테고리 보러갈래! 라고 할 수 있습니다..

그래서 저희는 로봇이 스스로 전시물을 인식하고 시맨틱 맵을 생성한 뒤,
사용자가 색상만 선택하면 해당 작품까지 안내하는 시스템을 구현했습니다.



#. 2장 Problem

먼저 저희가 왜 이런 방향으로 프로젝트르 생각했냐.
저희는 자율주행 시스템에서 단일 센서만 사용하는 방식의 한계에 관심이 있었습니다.

대표적인 예로, 카메라 기반 자율주행 차량은 실제 도로가 아닌 도로 사진이나 현수막을 실제 도로로 오인하는 사례가 알려져 있습니다.

반대로 LiDAR만 사용하는 경우에는 물체까지의 거리와 형태는 알 수 있지만, 그것이 무엇인지는 알기 어렵습니다.
즉, 카메라는 의미를 잘 이해하지만 거리 정보가 부족하고,
LiDAR는 거리 정보는 정확하지만 의미를 이해하기 어렵습니다.

그래서 저희는 두 센서를 함께 활용하여,
"무엇이 있는지"와 "어디에 있는지"를 동시에 저장할 수 있는 시스템을 만들어 보고자 했습니다.

이것이 이번 프로젝트의 출발점이었습니다.

센서 융합이 실제로 의미 있는 예제가 뭐가 있을까 고민하다가 미술관을 선택했습니다.
미술관은 넓고 전시물이 많기 때문에 원하는 작품을 찾기 어려울 수 있습니다.
특히 전시가 바뀔 때마다 작품 위치도 함께 변경됩니다.

만약 모든 작품 위치를 사람이 직접 등록해야 한다면 관리 비용이 커집니다.
또 실제 환경에서는 관람객이나 장애물도 계속 움직입니다.



# 3장 solution
그래서 저희는 매우 단순한 방식을 제안합니다.
사용자는 원하는 전시 구역의 색상을 선택합니다.
그러면 시스템은 미리 생성해 둔 시맨틱 맵에서 해당 작품의 위치를 찾고,
Nav2를 이용해 목적지까지 자율주행합니다.
도착 후에는 작품을 정면으로 바라보도록 정렬하고,
관람객은 원하는 카테고리의 작품을 순차적으로 감상할 수 있습니다.

누군가 뭘 하는게 쉬워 보인다면 그건 고수인것이다 라는 말이 있죠.
저희는 이를 구현하기 위해
카메라와 LiDAR의 장점을 결합한 시맨틱 내비게이션 방식을 적용했습니다.
카메라는 작품의 색상과 번호를 인식하여 "무엇인지"를 판단하고,
LiDAR와 SLAM은 "어디에 있는지"를 계산합니다.
이 두 정보를 결합하면 단순한 지도 생성이 아니라,
작품의 의미 정보까지 포함된 시맨틱 맵을 만들 수 있습니다.
사용자는 좌표를 입력할 필요 없이, 
"초록색 작품을 보고 싶다" 와 같이 의미 단위로 목적지를 선택할 수 있습니다.
시스템은 시맨틱 맵에서 해당 작품의 위치를 찾고, Nav2를 이용하여 작품 앞까지 자율주행합니다.

즉, 저희 프로젝트의 핵심은 센서 융합을 통해 의미 정보와 위치 정보를 함께 저장하고,
이를 실제 자율주행에 활용하는 것입니다.





# 4장 사용자 시나리오

사용자 흐름을 간단히 설명하자면
먼저 사용자가 보고 싶은 색상 구역을 선택합니다.

그 다음 시스템은 저장된 지도에서 해당 작품 위치를 검색합니다.
이후 Nav2가 경로를 생성하고 목적지까지 이동합니다.
도착 후에는 작품 앞에 정렬하여 멈추고, 설명을 제공할 수 있겠지요.

사용자 입장에서는 단순히 색상만 선택하면 되지만,
그 뒤에서는 시맨틱 맵 조회와 자율주행이 자동으로 수행됩니다.




# 5장 시스템 구성

프로젝트에 사용한 주요 구성 요소들입니다.
저희는 매핑의 실현에 있어서 가능성을 점칠 수 없었으므로, 
먼저 Gazebo 환경에서 전시관을 구성하고, 시뮬레이팅을 먼저 돌린 후에 현실에서 적용했습니다.
로봇과 기본 기술은 저희 수업에서 배운 turtlebot SLAM Toolbox를 기반했고,

그리고 사용자가 선택한 작품 위치까지 이동하기 위해
AMCL로 현재 위치를 추정하고,
Nav2를 이용해 경로 생성과 자율주행을 수행했습니다.

작품 인식은 카메라 영상을 기반으로 수행했습니다.
OpenCV를 이용해 작품의 색상을 구분하고,
EasyOCR을 이용해 작품 번호를 읽었습니다.

또한 LiDAR와 카메라에서 얻은 정보를
TF 좌표계를 이용해 하나의 지도 좌표로 변환했습니다.

그럼 이 시스템이 실제 어떤 구조로 동작하는지 한번 확인해 보겠습니다.



# 6장 핵심 아이디어

이 프로젝트는 운영을 두 단계로 나눴습니다.
매핑모드와 서비스 모드.

첫 번째는 매핑 모드입니다.
전시관이 운영되지 않는, 노이즈가 최대한 적은 시간에
로봇이 스스로 공간을 탐색하면서 
지도와 작품 정보를 수집합니다.

두 번째는 서비스 모드입니다.
사용자가 방문했을 때 이미 저장된 정보를 활용하여 안내를 수행합니다.

이렇게 두 단계로 나눈 이유는
환경 안에 존재하는 정보를 두 종류로 구분했기 때문입니다.

작품의 위치처럼 비교적 고정된 정보는
미리 수집해서 저장할 수 있습니다.

반면 관람객이나 갑자기 나타나는 장애물은
사전에 알 수 없기 때문에
주행 중 실시간으로 처리해야 합니다.

이제 첫번째 단계인 매핑 모드부터 설명드리겠습니다.

 
# 7장 매핑 모드

매핑 모드의 목표는
단순히 지도를 만드는 것이 아니라,
작품의 의미 정보와 위치 정보를 함께 저장하는 것입니다.

먼저 로봇은 전시관을 탐색하면서
SLAM Toolbox를 이용해 점유격자지도를 생성합니다.

동시에 카메라를 통해 작품의 색상과 번호를 인식합니다.
하지만 색상과 번호만 인식해서는
실제 위치를 알 수 없기에 LiDAR 거리 정보와 TF 좌표 변환을 이용하여
실제 지도 좌표로 저장했습니다.
같은 작품을 여러 번 발견할 수 있기 때문에
중복 등록을 제거하는 과정도 수행했습니다.

최종적으로 두 개의 결과물.
Nav2가 사용하는 점유격자맵과 작품의 색상, 번호, 좌표를 저장하는 시맨틱 맵입니다.

그래서 두가지의 결과물이 무슨 뜻인가 하면요.



# 8장 두가지 지도.
저희는 두 종류의 지도를 사용합니다.

첫 번째는 Occupancy Grid Map입니다. 점유격자 맵이죠.

SLAM Toolbox가 생성된 지도이며,
벽과 장애물, 이동 가능한 공간 정보를 포함합니다.

두 번째는 Semantic Map입니다.
작품의 색상, 번호, 위치 정보를 저장합니다.


즉, Occupancy Grid Map은 어디로 갈 수 있는지를 알려주고,
Semantic Map은 무엇이 어디에 있는지를 알려주는 역할을 합니다.

이 작업이정말... 힘들었습니다.
현실에서 camera로 읽은 객체의 데이터 맵과와 센서로 받은맵을 일치시키는거..
정말 ... 절망을 잠시 보고 왔습니다.








#9장 ocr hsv엔진

그렇다면 Semantic Map은 어떻게 생성될까요?
저희는 작품을 구분하기 위해 두 가지 정보를 사용했습니다.

첫 번째는 색상입니다.
OpenCV HSV 기반으로 빨강, 초록, 파랑을 구분합니다.

미술관의 예시 안에서 보면 빨강은 모네, 초록은 피카소, 파랑은 램브란트가 될 수 있겠죠.
하지만 색상만으로는 같은 구역 안의 여러 작품을 구별해 순차적으로 못가겠죠.

그래서 두 번째, 작품 번호를 도입했습니다.
미술관에 왔으면, 피카소의 그림을 연도순으로 보고싶을 수 있잖아요?
EasyOCR을 이용해 패널에 적힌 숫자를 읽어 작품을 식별합니다.

아주 오랜 고생 끝에
초록색 1번,초록색 2번, 초록색 3번 작품을 각각 다른 객체로 저장할 수 있었습니다.

또 OCR은 연산량이 크기 때문에
항상 수행하지 않고, 패널이 충분히 가까워졌을 때만 실행하여 성능을 최적화했습니다.

최종적으로 하나의 작품은
색상, 번호, 좌표 세 가지 정보를 가지고 맵에 저장됩니다.

# 10장 완성된 지도

매핑이 끝나면 최종적으로 이런 형태의 정보가 생성됩니다.
앞에서 설명드린 것처럼

저희는 단순히 벽만 그린 것이 아니라,
작품 정보까지 함께 저장했습니다.... 만
map 토픽에서 바로 시각적으로 보기까지는 좀 힘들었고,
시맨틱 맵은 yaml파일로 따로 좌표계가 저장이 됩니다.

이제부터는 이 정보를 이용해
실제로 사용자를 안내하게 됩니다.





# 11장 서비스 모드
이제 2번 단계. 서비스 모드입니다.

사용자가 초록1 작품을 선택했다고 가정해보겠습니다.
시스템은 시맨틱 맵에서 초록1 작품의 위치들을 검색하고
그 좌표를 Nav2 목표 지점으로 전달합니다.

이 후에는
저희가 수업에서 배운것과 동일하게,
AMCL이 현재 위치를 추정하고,
Nav2가 경로를 생성하여 목적지까지 이동합니다.
물론 장애물 발생 시 Nav2 Costmap을 통해 경로를 재계산하고 회피할 수 있습니다.

12장 작품 순회 및 정렬

단일 목적지 이동뿐 아니라 도슨트투어 모드도 구현했습니다.

미술관에 가서 하나만 보고오는 게 아니라
큐레이팅 투어를 받을 수 있는거죠.

예를 들어 초록색 구역을 선택하면
1번, 2번, 3번 작품을 순서대로 방문하고,
각 작품에서 일정 시간 머문 뒤 다음 작품으로 이동합니다.
db에 이미 작품의 설명이 삽입되어 있거나 다른 api연결시,
자동으로 작품 설명을 재생할 수 있겠죠.

최종적으로는 작품 정면을 바라보도록 정렬한 상태에서 정지합니다.

# 14장 확장성

이 시스템은 미술관 외에도 다양한 실내 환경에 적용 가능합니다.
색상은 구역을 구분하기 위한 하나의 예시입니다.

병원에서는 진료과 안내,공항에서는 게이트 안내,
대형마트에서는 매장 안내, 도서관에서는 서가 안내에 활용할 수 있습니다.

환경이 바뀌더라도 한 번 매핑만 수행하면 새로운 환경에 적응할 수 있다는 장점이 있죠.

중요한 것은 색상이 아니라
의미 정보를 위치 정보와 연결하는 방식입니다.

귀한 시간 집중해주셔서 감사하고
