Gazebo + TurtleBot3 quick reference for agents.
Simulation is the first verification surface; hardware is second.

Commands
- export TURTLEBOT3_MODEL=burger          # required before launching
- ros2 launch turtlebot3_gazebo turtlebot3_world.launch.py
- ros2 launch turtlebot3_gazebo empty_world.launch.py
- ros2 run turtlebot3_teleop teleop_keyboard          # manual drive
- ros2 launch <pkg> <launch_file> world:=<path>.world  # custom world

Constraints
- Set TURTLEBOT3_MODEL=burger; other models change footprint and sensor layout.
- Custom worlds and maps for the capstone live in pkgs/capstone_color_maze.
- The simulated camera faces the robot's heading, same as the real Burger.

Pitfalls
- Launching without TURTLEBOT3_MODEL set fails or spawns the wrong robot.
- Gazebo clock vs system clock: use use_sim_time:=true for nodes during simulation.
- Heavy worlds slow the physics step and distort sensor timing.
- World file paths must be absolute or correctly package-relative.
